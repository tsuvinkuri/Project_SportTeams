# План работы для выполнения заданий под набор данных варианта №7

## Анализ структуры csv файла и разработка набора классов-моделей (задания 1-3)
Для начала нужно ознакомиться с самим файлом .csv.
Откроем в Excel'е через вставку данных и проанализируем поля (какой тип данных для каждого нужен, есть ли шумы в данных).
Первое, что бросилось в глаза, хоть и не критичное - несоответствие ковычек для одних и тех же типов данных.
Позже в коде это все обрабатывалось через replac'ы.
Дальше по аналогии с тем, что ранее делали на практическом занятии, проанализировал структуру данных csv файла был реализован набор классов‑моделей.
Основным является класс SportTeams, в котором хранятся все атрибуты из CSV‑файла: имя игрока, команда, позиция, рост, вес и возраст. Для поля позиции был создан отдельный enum Position, так как количество позиций довольно сильно ограничено.
Далее был реализован класс Utilities, в котором находится метод для парсинга CSV‑файла и преобразования строк в объекты.
Чтение файла SportTeamsIndicators.csv было доработано и сделано постоянным, с указанием пути внутри проекта. 

## Разработка БД с использованием SQLite (задания 4-6)
На следующем этапе была настроена работа с базой данных SQLite.
В проекте создан класс DataBase, отвечающий за:
подключение к базе SportTeams.db3;
создание таблицы SportTeams в случае её отсутствия;
запись всех объектов из обработанного файла csv в БД.
Структура таблицы SportTeams включает поля: уникальный идентификатор (добавлен на моменте создания таблицы для того, чтобы это поле было содержало первичный ключ), имя игрока, команда, позиция, рост, вес и возраст.

## Разработка и выполнение SQL-запросов (задания 7-9)
После заполнения базы данных был реализован класс Analytics.
В нем выполняются следующие действия:
- открывается соединение с БД (по этой причине в Main соединение было помещено в комментарий, чтобы не открывать соединение дважды);
- выполняется набор SQL-запросов в соответствии с заданием;
- результаты выводятся в консоль;
- визуализируются данные с помощью библиотеки JFreeChart.
Реализованные SQL-запросы включают:
- расчёт среднего возраста игроков по командам;
- определение команды с максимальным средним ростом;
- выборку пяти самых высоких игроков ранее выбраной команды;
- поиск команды с максимальным средним возрастом среди игроков с ростом в диапазоне 74-78 дюймов и весом 190-210 фунтов.
